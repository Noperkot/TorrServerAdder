This extension allows you to add torrents from torrent trackers to TorrServer and play them using the system media player.

For the extension to work, !!! YOU NEED A TORRSERVER !!! running on a local machine or on a network (for example, from a TorrServe client application in Android or using TorrServer Launcher in Windows).

Features

  • Functions of the context menu (RIGHT click on the torrent or the magnet link):
    Add to TorrServer — the torrent will be added to the server and will be available for playback by clients.
    Add and play — the torrent is also added to the server, after which the browser downloads a playlist of the added torrent from the server and opens it in the PC media player for playback.
    Play without adding — similar to the previous command, but after playing the torrent will be deleted from the server.
  • Catch a magnet links (by clicking on the link, the action specified in the add-on settings is performed). The catching can be temporarily disabled by pressing Ctrl key.
  • Together with the torrent, its poster and name are transmitted to the server if possible (only for specific torrent trackers).

    To send a poster you need to add a torrent from the torrent page. TorrServer 1.1 TorrServer 1.1 accepts posters sent only by MAGNET link.

    The poster will be visible from clients (TorrServe, Kodi TorrServer Client), and in the TorrServer MatriX(1.2) web interface.

    List of torrent-trackers that support the transfer of posters:

    * updatable torrents support

    The search for torrent updates is launched from the profiles menu (LMB on the extension icon), and is performed only for torrents that meet the following requirements:

    • The torrent must be added using the extension "__MNF_name__" version at least 2.0.0
    • Torrent must be added from a supported tracker (*)
    • The torrent must be added from his personal page
    • When using TorrServer 1.1.x, the torrent must be added via magnet link
    • Checking for updates to torrents added from KINOZAL.TV is possible only when authorized on the service. Otherwise, a home page error will be thrown.

    When updating the torrent, the transfer of the marks about the watched episodes is carried out only for the MatriX server. For some torrents, the transfer may not occur correctly. In this case, the marks can be placed manually.

    On trackers covered by Cloudflare, when searching for updates, errors can sometimes occur (5xx or home page unavailable). In such cases, you can try:

    • Restart the search for updates for all torrents by refreshing the browser page
    • Run search for erroneous torrents one by one
    • Open the torrent page for checking by the browser service, and then restart the general search for updates
  • Profile system. Ability to store multiple sets of settings. Switching profiles is performed from the menu called by left-clicking on the extension icon.
  • Authorization support.
  • On the torrent page in KINOZAL.TV, a magnet link is displayed (only for registered users).
  • Support for updatable torrents (added from trackers marked with * in the list of torrent trackers).
  • Clearing traces in the browser's download history.
  • Supported TorrServer versions: MatriX, 1.2, 1.1.

Settings

  • Make sure that TorrServer is running and opens its web interface in the browser. The latest version of the server for your platform can be found here.
  • Playback requires your media player to be associated with ".m3u" files.
  • After extension installing, specify the network address of your TorrServer like http://torrserver.lan:8090 or http://localhost:8090 (if the server is running on the same host as the browser).
    You can use an IP address instead of a name (http://192.168.1.123:8090).
    For authorization (if used by the server), the url must contain a name and password (for example, http://username:password@192.168.1.123:8090).
  • Select the action to perform when you click on the magnet link.
  • Set the clear download list flag only after you make sure that the player picks up the playlist without prompting the browser.
  • To prevent a browser confirmation to open a playlist file, you must:

    Firefox ver. ≥ 98:

    • after the first download (playback), in the browser downloads list in the context menu (right mouse button on the downloaded .m3u file), select "Always open files of this type" (the location of this option may vary depending on the browser version).

    Firefox ver. < 98:

    • install the InlineDisposition Reloaded extension. The first couple of times when starting playback, the browser may still ask for confirmation — to agree. Or ask in which application to open the file — choose your media player.

    • Additional settings for Tor Browser:

  • The TorrServer address must be an IP address.
  • You need to disable the Tor proxy for the TorrServer:
    • about:config ⇒ insert network.proxy.no_proxies_on in the search bar ⇒ set the IP address of your TorrServer (x.x.x.x for the address on the network or 127.0.0.1 if the server is running on the same host as the browser). After this operation, the server page should be available in the browser (http://x.x.x.x:8090)
