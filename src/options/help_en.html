This extension allows you to add torrents from torrent trackers to TorrServer and play them using the system media player.


Features

  • Functions of the context menu (RIGHT click on the torrent or the magnet link):
    • Add to TorrServer — the torrent will be added to the server and will be available for playback by clients.
    • Add and play — the torrent is also added to the server, after which the browser downloads a playlist of the added torrent from the server and opens it in the PC media player for playback.
    • Play without adding — similar to the previous command, but after playing the torrent will be deleted from the server.
  • Catch a magnet links (by clicking on the link, the action specified in the add-on settings is performed)
  • A poster is passed along with the torrent (only for specific torrent trackers)
    To send a poster you need to add a torrent from the torrent page. TorrServer 1.1 accepts posters sent only by MAGNET link.
    The poster will be visible from clients (TorrServe, Kodi TorrServer Client), and in the TorrServer MatriX(1.2) web interface.
    If the Ctrl key was pressed at the time of adding the torrent, the url of the page from which it was taken will be added to the torrent name (Only for MatriX. The url can be seen in the "INFO" of the torrent in the web interface of the TorrServer).
    The availability of the poster depends on many factors (from changing the page layout to various locks), so no guarantees.

    List of trackers that support the transfer of posters:

    • Profile system. Ability to store multiple sets of settings. Switching profiles is performed from the menu called by left-clicking on the extension icon
    • Authorization support
    • On the torrent page in KINOZAL.TV, a magnet link is displayed (only for registered users)
    • Clearing traces in the browser's download history
    • Supported TorrServer versions: MatriX, 1.2, 1.1

    Settings

    • After extension installing, specify the network address of your TorrServer like http://torrserver.lan:8090 or http://localhost:8090 (if the server is running on the same host as the browser). You can use an IP address instead of a name (http://192.168.1.123:8090).
    • Select the action to perform when you click on the magnet link.
    • Set the clear download list flag only after you make sure that the player picks up the playlist without prompting the browser.
    • o that the browser does not ask for confirmation to open the playlist file every time, after the first download (playback) in the list of browser downloads, you must check the "Always open files of this type" box on the downloaded .m3u file.
    • To prevent the browser from requesting confirmation to open the playlist file every time, you need to install the InlineDisposition Reloaded add-on. The first couple of times when starting playback, the browser may still ask for confirmation-to agree. Or ask in which application to open the file — choose your media player.

      • Additional settings for Tor Browser:
    • The TorrServer address must be an IP address.
    • You need to disable the proxy for the server — about:config -> insert network.proxy.no_proxies_on in the search bar - > set the IP address of your TorrServer (x.x.x.x for the address on the network or 127.0.0.1 if the server is running on the same host as the browser). After this operation, the server page should be available in the browser (http://x.x.x.x:8090)
